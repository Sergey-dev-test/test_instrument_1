# Настройка ИИ-помощника

## Быстрый старт

### 1. Откройте настройки ИИ
В правом верхнем углу приложения нажмите иконку ⚙️ (шестерёнка) или 🤖.

### 2. Выберите провайдера

#### OpenAI (GPT)
1. Выберите "OpenAI (GPT)" из списка
2. Вставьте API ключ (получить на https://platform.openai.com/api-keys)
3. Выберите модель (gpt-3.5-turbo, gpt-4, gpt-4-turbo)
4. Нажмите "Проверить подключение"
5. Сохраните настройки

#### Anthropic (Claude)
1. Выберите "Anthropic (Claude)"
2. Вставьте API ключ (получить на https://console.anthropic.com/)
3. Выберите модель (claude-3-haiku, claude-3-sonnet, claude-3-opus)
4. Нажмите "Проверить подключение"
5. Сохраните настройки

#### Azure OpenAI
1. Выберите "Azure OpenAI"
2. Введите URL в формате: `https://your-resource.openai.azure.com/openai/deployments/your-deployment/chat/completions?api-version=2023-05-15`
3. Вставьте API ключ
4. Введите имя деплоймента
5. Нажмите "Проверить подключение"
6. Сохраните настройки

#### Свой API
1. Выберите "Свой API"
2. Введите URL вашего API
3. Вставьте API ключ
4. Введите имя модели
5. Нажмите "Проверить подключение"
6. Сохраните настройки

## Настройки

### Temperature (0-2)
Контролирует креативность ответов:
- 0.0 - максимальная точность
- 0.7 - баланс точности и креативности
- 1.0+ - высокая креативность

### Max Tokens
Максимальная длина ответа в токенах:
- 256 - короткие ответы
- 1024 - средние ответы
- 2048 - подробные ответы
- 4096 - очень длинные ответы

### System Prompt
Инструкция для ИИ о том, как себя вести. По умолчанию:
```
Ты - помощник по работе со справочными данными. Помогай пользователю анализировать, создавать и редактировать данные в таблицах.
```

Можно настроить под ваши задачи:
```
Ты - эксперт по PostgreSQL. Помогай писать SQL-запросы, оптимизировать таблицы и анализировать данные.
```

## Чат с ИИ

### Открытие чата
Нажмите иконку 🤖 в верхней панели.

### Возможности
- 💬 Отправка сообщений (Enter для отправки)
- ⏹️ Отмена генерации
- 📋 Копирование ответов
- 🗑️ Очистка истории
- 💾 Автосохранение истории в localStorage

### Примеры вопросов
- "Какие таблицы есть в базе данных?"
- "Напиши SQL для выборки всех сотрудников из отдела IT"
- "Проанализируй структуру таблицы employees"
- "Создай новый запрос для экспорта данных"

## Решение проблем

### Ошибка "API ключ недействителен"
- Проверьте правильность API ключа
- Убедитесь, что ключ активен
- Проверьте лимиты использования

### Ошибка "Модель не найдена"
- Проверьте имя модели
- Убедитесь, что модель доступна в вашем регионе
- Проверьте права доступа к модели

### Ошибка "Timeout"
- Увеличьте время ожидания
- Проверьте стабильность интернет-соединения
- Попробуйте другую модель (более быструю)

### Чат не сохраняется
- Проверьте, включены ли cookies/localStorage в браузере
- Попробуйте очистить кэш браузера
- Проверьте настройки приватности

## Безопасность

- API ключи хранятся локально в браузере (localStorage)
- Ключи не передаются на сервер Draft Tool
- Все запросы к ИИ-провайдерам идут напрямую
- Рекомендуется не использовать ключи с высокими лимитами

## Поддержка

Если возникли проблемы:
1. Проверьте логи консоли браузера (F12)
2. Убедитесь, что API ключ действителен
3. Проверьте интернет-соединение
4. Попробуйте очистить настройки ИИ и настроить заново
