# ТЕХНИЧЕСКОЕ ЗАДАНИЕ
# Приложение для управления структурой баз данных с ИИ-интеграцией

## 1. ОБЩИЕ СВЕДЕНИЯ

### 1.1. Наименование системы
Инструмент управления структурой баз данных с автоматической генерацией SQL и методик ведения таблиц.

### 1.2. Назначение системы
- Подключение к различным СУБД
- Автоматический анализ структуры баз данных
- Генерация SQL-запросов
- Генерация методик ведения таблиц
- Работа с ИИ-агентом через API
- Ролевой доступ к функциям

### 1.3. Целевые пользователи
- Администраторы баз данных
- Разработчики
- Аналитики данных
- Руководители проектов

## 2. ТРЕБОВАНИЯ К СИСТЕМЕ

### 2.1. Функциональные требования

#### 2.1.1. Подключение к базе данных
- Ручной ввод параметров подключения (хост, порт, имя БД, логин, пароль)
- Автоматический поиск доступных баз данных на локальном компьютере
- Поддержка СУБД: PostgreSQL, MySQL, SQLite, SQL Server
- Тестирование подключения перед сохранением

#### 2.1.2. Анализ структуры БД
- Автоматическое определение всех таблиц
- Определение полей таблиц (имя, тип, ограничения)
- Определение первичных ключей
- Определение внешних ключей и связей
- Визуализация связей в виде графа/ER-диаграммы

#### 2.1.3. Генерация SQL-запросов
- Автоматическая генерация SELECT-запросов
- Генерация INSERT-запросов для новых записей
- Генерация CREATE TABLE на основе структуры
- Предпросмотр и редактирование сгенерированных запросов
- Валидация SQL перед выполнением

#### 2.1.4. Генерация методик ведения таблиц
- Автоматическое формирование описания таблицы:
  * Наименование таблицы
  * Владелец таблицы
  * Список полей с типами
  * Описание связей
- Экспорт методик в текстовом формате
- Экспорт в PDF

#### 2.1.5. Управление данными
- **Запрещено:**
  * Удаление существующих записей
  * Изменение существующих данных
- **Разрешено:**
  * Создание новых таблиц
  * Добавление новых записей
  * Загрузка данных из Excel (.xlsx, .xls)

#### 2.1.6. Интеграция с ИИ-агентом
- REST API для подключения внешних ИИ-агентов
- Отправка запросов на анализ данных
- Получение рекомендаций от ИИ
- Чат-интерфейс для взаимодействия

#### 2.1.7. Ролевая модель доступа
- **Администратор:**
  * Полный доступ ко всем таблицам
  * Редактирование и удаление любых таблиц
  * Управление пользователями
  * Настройка системы
- **Пользователь:**
  * Создание новых таблиц
  * Редактирование только своих таблиц
  * Удаление только своих таблиц
  * Просмотр всех таблиц

### 2.2. Нефункциональные требования

#### 2.2.1. Масштабируемость
- Горизонтальное масштабирование (добавление серверов)
- Вертикальное масштабирование (увеличение ресурсов)
- Поддержка множественных подключений к БД
- Кэширование структуры БД

#### 2.2.2. Производительность
- Время отклика интерфейса: до 2 секунд
- Анализ структуры БД до 1000 таблиц: до 30 секунд
- Параллельная обработка запросов

#### 2.2.3. Безопасность
- Шифрование паролей при хранении
- Аутентификация пользователей
- Авторизация на основе ролей
- Логирование всех действий
- Защита от SQL-инъекций

#### 2.2.4. Совместимость
- ОС: Windows 10+, Linux, macOS
- Браузеры: Chrome, Firefox, Edge, Safari
- Форматы Excel: .xlsx, .xls

## 3. ТЕХНОЛОГИЧЕСКИЙ СТЕК

### 3.1. Бэкенд
- Язык: Python 3.10+
- Фреймворк: FastAPI
- ORM: SQLAlchemy
- СУБД проекта: PostgreSQL (для хранения метаданных)
- Очередь задач: Celery + Redis
- Документация API: Swagger/OpenAPI

### 3.2. Фронтенд
- Фреймворк: React 18+ или Vue 3
- Язык: TypeScript
- UI-библиотека: Ant Design / Material-UI
- Визуализация связей: React Flow / D3.js
- Загрузка Excel: xlsx.js

### 3.3. Инфраструктура
- Контейнеризация: Docker, Docker Compose
- Оркестрация: Kubernetes (опционально)
- CI/CD: GitHub Actions / GitLab CI
- Мониторинг: Prometheus + Grafana
- Логирование: ELK Stack

## 4. ТРЕБОВАНИЯ К ИНТЕРФЕЙСУ

### 4.1. Приветственное окно
- Поле ввода адреса БД
- Кнопка "Поиск БД"
- Кнопка "Подключиться"
- Список найденных БД (если поиск успешен)

### 4.2. Основной интерфейс
- Дерево таблиц и связей (левая панель)
- Область просмотра/редактирования (центральная)
- Панель инструментов (верхняя)
- Панель статуса (нижняя)

### 4.3. Модуль генерации SQL
- Визуальный конструктор запросов
- Текстовое поле для ручного редактирования
- Кнопка "Выполнить"
- Результат выполнения

### 4.4. Модуль методик
- Предпросмотр методики
- Кнопки экспорта (TXT, PDF)

### 4.5. Чат с ИИ-агентом
- Область сообщений
- Поле ввода запроса
- Индикатор загрузки

## 5. ТРЕБОВАНИЯ К БЕЗОПАСНОСТИ ДАННЫХ

1. Все операции изменения/удаления существующих данных заблокированы на уровне UI и бэкенда
2. Только пользователи с ролью "Администратор" могут выполнять изменения
3. Все операции логируются с указанием пользователя, времени и действий
4. Пароли от БД хранятся в зашифрованном виде
